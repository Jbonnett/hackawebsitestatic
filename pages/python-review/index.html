What is this Document

  • Basic Notes

    • Scope is defined by indentation level, no brackets.
    • OO, procedural and functional programming are all supported well.
    • So is meta programming and introspection.
    • No type based polymorphism of functions
    • memory management is mostly automatic.
    • __ indicates internal structure
    • __init__ is constructor
    • Range is non inclusive on the end value
    • print requires parentheses now, because it is a function not a statement
    • if, for, and while are statements and DO NOT REQUIRE PARENTHESES around the condition
    • list comprehensions are great for pre allocating [x for x in range(6) if x > 0] = [1,2,3,4,5]

  • Operators

    • ** is square
        • / * all work on ints as expected, Ints are now unbounded and can be however big (#python3ism )
    • or, and, not are all texual
    • del(somevalue) seems pretty universal to free memory
    • operators have per class overrides you can provide with __operatorname__

  • Data Structures

    {{video https://www.youtube.com/watch?v=kQDxmjfkIKY}} video is unrelated but seems good

    • Lists

      • [] literal notation, list() class
      • item in list rather than .contains() or .hasItem() #python3isms
      • list.insert(item) list.insert(items.)
      • list.extend(list of items),
      • list.remove(item) similar to del(list[index]) but looks by item
      • list.sort() sorts list in place
      • list.sorted() returns a sorted list.
      • list.reverse() in place reverse
      • list.pop() return last item and remove from list

    • Tuples

      • immuteable list
      • much faster than lists
      • () literal notation, but just a series with coma’s works, also tuple()
      • objects inside tuples can be mutable

    • Sets

      • unordered groups of unique items
      • {} is the literal notation, set() class
      • & | ^ - all work as expected
        • | union
        • ^ xor
          • subtraction (remove items)
      • <= and >= work to tell if subset/super set

    • Dictionaries

      • unsorted hash maps
      • {} literal notation with colon separated pairs, dict() class notation, takes a list of tuples or kwargs
      • [‘key’] to lookup and assign similar to a string list
      • key in dict rather than .hasKey(‘key’) though the later works
      • .items() returns a list of tuples
      • .keys() returns list of keys
      • .values() returns list of values

    • ByteArrays

      • Use instead of strings for byte manipulation in python 3 #python3isms
      • Single Bytes can be represented as b"\xA9" etc
      • you can also use a Array() of [[type]] “B” for unsigned char for a closer to C implementation

    • Stacks

      • last in first out sequence
      • append() (read push()), pop(), clear() all work as expected
      • peek() has to be implemented as [-1]

    • Queues

      • same thing but fifo
      • python has a imp in collections

    • MaxHeap

      • Binary tree, ** note not Binary Search Tree **
      • Very fast, gets the max item super fast, remove max fast.
      • Basically just a list and indexes that go up by multiples of 2 as you go down the tree.
      • left child index = index *2
      • right child index = index * 2 +1
      • parent index = index/2
      • pop and peak all work on max
      • push puts item on end of array then floats up to right part in tree, can be kinda slow in worst case.

    • Binary Search Trees

      • A given node is more than items in left sub tree and more than all items in right sub tree
      • .insert(item) as a leaf, but find where by walking down tree
      • .delete(item) is complicated. Has several diff options item is leaf, item has 1 child, item has 2 items
        • 1 child promotes single child whole sub tree
        • if 2 children, get left most node in right sub tree and swap deleted item.
      • .get_size() recursively get size as 1 + size left sub tree + size right sub tree
      • .traversal() options, pre order, in order
        • .pre_order() is root, then left, then right,
        • .in_order() is bottom up and to the right, returning a sorted list
      • BIG ADVANTAGE IS SPEED! O(log n) in a blanced tree

    • Graphs

      • Made up of nodes and edges that connect the nodes/vertexes

      • Undirected graphs

        • Bidirectional connections between nodes

      • Directed Graph

        • Directional connections only

      • Implementation Options

        • Adjacency List

          • list stored in each vertex
          • slower for dense graphs

        • Adjacency Matrix

          • 2d matrix of neighbors stored central in graph object
          • from and to are x and y axis of the matrix with a 0 or false representing non connection and a one or true representing connection. Mirrored over the diagonal in a undirected. Weights easy to store instead.
          • Matrix better for dense graphs

        • List of edge tuples

          • non oo representation
          • kinda a pain in the ass, might be worth converting to oo for any operations

    • List convenience functions

    • Generators

      • Functions that implement the sequence api and let you yield a result each time they are asked for another number. Useful for saving memory by generating 1 value at a time, with the syntactic sugar of being able to treat it as a sequence (like a list)

    • Except has a finally statement

    • Named Tuples

      • part of collections
